═══ Oh My World — Formula Writing Guide ═══

1. Format

Separate layers with semicolons ";". Two formats are supported:

[Single expression]
  y=<start>..<end>: <expression>
  y=<layer>: <expression>             ← single layer, range omitted

[let block]
  y=<start>..<end>: {
    let var = <expression>;
    let var = <expression>;
    ...
    <final expression>
  }

  Variables declared with 'let' are scoped to the { } block.
  Semicolons inside { } are statement separators, not layer separators.

[Cyclic expression]
  y=<start>..<end>: <count>*[<expr>],<count>*[<expr>],...

  Within the specified y-range, the comma-separated groups repeat
  in cycles until the entire range is filled.

Default preset:
  y=0: minecraft:bedrock;y=1..64: (x+z)%2==0 ? minecraft:white_concrete : minecraft:gray_concrete


2. Variables
  x    World X coordinate (integer, can be negative)
  z    World Z coordinate (integer, can be negative)
  ly   Relative y within the layer (0-based, ly = currentY - layerStartY)


3. Operators
  Arithmetic: + - * / %
  Comparison: == != < > <= >=
  Logical:    && || !
  Grouping:   ( )
  Ternary:    condition ? valueA : valueB


4. Built-in Functions
  Function call syntax: functionName(arg1, arg2, ...)

  -- Integer --
  floordiv(a, b)  Floor division (toward -infinity)
  floormod(a, b)  Floor modulo (result always >= 0)

  -- General Math --
  abs, max, min, floor, ceil, round, sign, sqrt, pow, exp, log, log10

  -- Trigonometric (radians) --
  sin, cos, tan, asin, acos, atan, todeg, torad

  -- Random --
  rand()                Random block from ALL registered blocks (excluding air)
  rand(b1, b2, ...)     Whitelist — random from the specified blocks
  randexcept(b1, b2, ...) Blacklist — random from all blocks EXCEPT specified

  ★ Use floordiv/floormod instead of / and % for grid patterns.
    rand() is deterministic: same (x,z,y) always returns the same block.


5. Expression Examples
  ▸ Uniform: y=0: minecraft:bedrock
  ▸ 1×1 checker: y=1..64: (x+z)%2==0 ? minecraft:white_concrete : minecraft:gray_concrete
  ▸ 3×3 grid: y=1..64: (floordiv(x,3)+floordiv(z,3))%2==0 ? minecraft:white_concrete : minecraft:gray_concrete
  ▸ Stripes: y=1..64: floordiv(x,4)%2==0 ? minecraft:white_concrete : minecraft:gray_concrete
  ▸ 3D model: y=0..64: (x+z)%2==0 && ly<10 ? minecraft:white_concrete : minecraft:air

  ▸ let block — avoid recomputing sub-expressions:

    y=-64..319: {
      let cellX = floordiv(x, 192);
      let cellZ = floordiv(z, 192);
      let seed = cellX*37.1 + cellZ*91.7;
      let offX = sin(seed)*40;
      let offZ = cos(seed*1.3)*40;
      let dx = x - (cellX*192 + 96 + offX);
      let dz = z - (cellZ*192 + 96 + offZ);
      dx*dx + dz*dz < 400 ? minecraft:white_concrete : minecraft:air
    }


6. Blocks
  Use standard Minecraft resource IDs.
  Supported concrete colors: white, gray, black, red, blue, yellow,
  green, orange, purple, light_gray, light_blue, magenta, lime, pink,
  cyan, brown.
  Other blocks are resolved via BuiltInRegistries.BLOCK.


7. Notes
  1. Use English semicolons ";" (not Chinese "；").
  2. All block IDs must be typed in English.
  3. Nest ternary conditions with explicit parentheses.
  4. For grid patterns, MUST use floordiv/floormod.
  5. "let" block semicolons are safe — they're statement separators,
     not layer separators.
